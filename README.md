# astrbot_plugin_ts3_tracker

一个基于 AstrBot 的 TeamSpeak 3 查询插件。

这个插件只保留了最基础的一项功能：当用户在 QQ 中发送 `上号` 时，机器人连接 TS3 ServerQuery，查询当前服务器状态，并返回在线人数和频道信息。

适合需要在 QQ 中快速查看 TS3 在线情况的场景，不包含持续监控、上线下线通知、数据存储等额外功能。

## 功能说明

用户发送 `上号` 后，机器人会返回以下内容：

- 服务器名称
- 服务器地址和端口
- 当前在线人数
- 当前频道列表
- 每个频道中的在线用户

这是一个精简版查询插件，目标是简单、直接、容易部署。

## 运行要求

- AstrBot `>=4.16,<5`
- 可正常连接到 TeamSpeak 3 ServerQuery 端口
- 已拥有可用的 ServerQuery 账号和密码

支持平台：

- `aiocqhttp`
- `qq_official`

## 安装方式

将插件目录放入 AstrBot 插件目录后，在 AstrBot WebUI 中加载或重载插件即可。

如果你使用 GitHub 管理插件，可以直接将本项目上传到自己的仓库，再通过 AstrBot 插件管理方式安装。

## 配置说明

在 AstrBot 插件配置页中填写以下项目：

### `server_host`

TS3 服务器地址。

示例：

```text
ts.example.com
```

或：

```text
127.0.0.1
```

### `server_port`

TS3 语音服务器端口。

默认常见值：

```text
9987
```

### `serverquery_port`

TS3 ServerQuery 端口。

默认常见值：

```text
10011
```

### `serverquery_username`

ServerQuery 登录账号。

### `serverquery_password`

ServerQuery 登录密码。

## 使用方法

插件加载完成并配置正确后，QQ 直接发送：

```text
上号
```

机器人就会返回当前 TS3 服务器信息。

## 项目特点

- 只保留一个核心功能
- 不做上线下线通知
- 不做历史记录和本地数据库
- 不做定时监控
- 适合想要最小可用插件的人直接使用

## 目录说明

当前项目只保留 AstrBot 插件运行所需的基础文件：

- `main.py`：插件主逻辑
- `_conf_schema.json`：AstrBot 插件配置定义
- `metadata.yaml`：插件元信息
- `__init__.py`：插件包入口

## 注意事项

- 机器人运行环境必须能访问 TS3 的 ServerQuery 端口。
- 如果 ServerQuery 账号密码错误，插件会返回查询失败信息。
- 如果 QQ 发送 `上号` 没有响应，请优先检查 AstrBot 插件是否加载成功，以及配置是否填写完整。
- 如果服务器网络有限制，请确认防火墙已放行对应端口。

## 适用场景

这个插件适合以下场景：

- 想在 QQ 里快速查看 TS3 是否有人在线
- 只需要手动查询，不需要持续推送通知
- 希望插件结构尽量简单，方便二次修改
